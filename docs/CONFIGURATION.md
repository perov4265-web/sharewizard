## Папка и SMB-ресурс

### Путь к папке
Абсолютный путь к папке, которую нужно опубликовать.  
Если папки не существует, программа создаст её автоматически (`os.makedirs`).

**Примеры:**
```
C:\SharedData\Проект
D:\Company\Reports
\\SERVER\Volume\Shared          ← UNC-путь тоже работает
```

### Имя SMB-ресурса
Сетевое имя ресурса — то, что пользователи будут вводить при подключении.

- Допустимые символы: латиница, цифры, дефис, подчёркивание
- Максимальная длина: 80 символов
- Символ `$` в конце делает ресурс скрытым (`\\PC\Hidden$`)

**Результат:** `\\ИМЯ_КОМПЬЮТЕРА\ИМЯ_РЕСУРСА`

### Описание
Отображается в сетевом окружении и в `Get-SmbShare`. Необязательное поле.

## Права доступа

Программа настраивает два независимых уровня прав.

### SMB-права (сетевые)
Применяются при подключении через сеть. Задаются командой `Grant-SmbShareAccess`.

| Уровень | Что разрешает |
|---------|--------------|
| **Read** | Только чтение файлов и папок |
| **Change** | Чтение + запись + удаление |
| **FullControl** | Всё, включая изменение прав |

> ⚠️ SMB-права — верхняя граница. Итоговые права = минимум(SMB, NTFS).

### NTFS-права (локальные)
Применяются к файловой системе через `FileSystemAccessRule`. Действуют и при локальном, и при сетевом доступе.

| Уровень | Что разрешает |
|---------|--------------|
| **Read** | Чтение файлов, просмотр папок |
| **Modify** | Чтение + запись + удаление (без изменения прав) |
| **FullControl** | Полный контроль, включая изменение NTFS-ACL |

### Наследование
При включённой опции «Наследование NTFS» права применяются ко всем вложенным папкам и файлам (`ContainerInherit, ObjectInherit`).

### Типовые конфигурации

**Только для чтения (архив):**
```
Everyone           → SMB: Read    / NTFS: Read
Administrators     → SMB: Full    / NTFS: Full
```

**Рабочая папка отдела:**
```
DOMAIN\Department  → SMB: Change  / NTFS: Modify
DOMAIN\Manager     → SMB: Full    / NTFS: Full
Administrators     → SMB: Full    / NTFS: Full
```

**Публичная (все пишут):**
```
Everyone           → SMB: Change  / NTFS: Modify
Administrators     → SMB: Full    / NTFS: Full
```

## Параметры SMB

### Access Based Enumeration (ABE)
Когда включено — пользователи видят только те папки и файлы, к которым у них есть доступ. Скрывает недоступные объекты из сетевого списка.

**Рекомендация:** включить для ресурсов с дифференцированными правами.

PowerShell-эквивалент:
```powershell
New-SmbShare -FolderEnumerationMode AccessBased
```

### SMB-шифрование
Шифрует сетевой трафик между клиентом и сервером (SMB 3.0+, AES-128).

**Требования:**
- Сервер: Windows Server 2012+ или Windows 8+
- Клиент: Windows 8+ / Windows Server 2012+

**Ограничение:** клиенты Windows 7 и XP не смогут подключиться.

### Максимум подключений
Лимит одновременных сессий к ресурсу. `0` = без ограничений.

### Кеширование файлов
Управляет режимом офлайн-доступа (Offline Files).

| Режим | Описание |
|-------|----------|
| **None** | Кеширование отключено (рекомендуется для серверных ресурсов) |
| **Manual** | Пользователь выбирает файлы для кеша вручную |
| **Programs** | Автоматическое кеширование исполняемых файлов |
| **Documents** | Автоматическое кеширование всех открытых файлов |

## Брандмауэр

### Порт 445 (SMB)
Основной порт протокола SMB (Server Message Block). Обязателен для доступа к сетевым папкам.

Команда:
```
netsh advfirewall firewall set rule group="File and Printer Sharing" new enable=Yes
Set-NetFirewallRule -DisplayGroup "File and Printer Sharing" -Enabled True -Profile Domain,Private
```

### Порты 137-139 (NetBIOS)
Порты устаревшего протокола NetBIOS over TCP/IP. Нужны для совместимости со старыми клиентами (Windows XP, некоторые сетевые принтеры).

> В современных сетях (только Windows 10/11) можно отключить.

### Сетевое обнаружение
Разрешает компьютеру «видеть» других участников сети и быть видимым самому.

Правила: группа `Network Discovery` в брандмауэре.

### Профиль брандмауэра

| Профиль | Применяется когда |
|---------|-------------------|
| **Domain + Private** | ПК в домене или в доверенной сети (рекомендуется) |
| **Any** | Все профили включая Public (небезопасно!) |
| **Domain** | Только доменные сети |
| **Private** | Только домашние/рабочие сети |

## Сеть и обнаружение

### Сетевое обнаружение
Запускает службы `fdPHost`, `FDResPub`, `SSDPSRV`, `upnphost` и прописывает настройку в реестр. Позволяет компьютеру появляться в «Сетевом окружении».

### Общий доступ к файлам и принтерам
Включает SMB v2 через реестровый ключ:
```
HKLM\SYSTEM\CurrentControlSet\Services\LanmanServer\Parameters\SMB2 = 1
```

### Разрешить доступ в публичных сетях
Открывает правила брандмауэра для профиля `Public`. **Использовать с осторожностью** — публичный профиль применяется в гостиницах, кафе и т.д.

### Отключить защиту паролем
Устанавливает `restrictnullsessaccess = 0`. Позволяет подключаться без авторизации. **Небезопасно** — используйте только во внутренних изолированных сетях.

## Службы и политики

### LanmanServer
Ключевая служба SMB-сервера. Без неё сетевые папки не работают. Программа устанавливает автозапуск и немедленно запускает службу.

### WinRM (Windows Remote Management)
Нужен для удалённого управления через PowerShell Remoting. Необязателен для базового SMB-доступа.

### Наследование NTFS-прав
`SetAccessRuleProtection($false, $true)` — включает наследование прав от родительской папки. Новые права добавляются поверх унаследованных.

### Аудит доступа
Записывает события в Windows Event Log (Security, Event ID 4663).

Настраивает:
- `FileSystemAuditRule` на папку (Everyone, FullControl, Success+Failure)
- Политику аудита: `auditpol /set /subcategory:"Object Access"`

Просмотр событий: `eventvwr.msc` → Windows Logs → Security.

### Лог в %TEMP%
Сохраняет текстовый журнал выполнения в папку временных файлов:
```
%TEMP%\ShareWizard_ИМЯ_РЕСУРСА_YYYYMMDD_HHMMSS.log
```

## Диагностика

Правая панель программы отображает текущее состояние системы.

| Индикатор | Как проверяется |
|-----------|----------------|
| Права администратора | `ctypes.windll.shell32.IsUserAnAdmin()` |
| LanmanServer | `sc query LanmanServer` |
| Брандмауэр SMB | `netsh advfirewall firewall show rule name=...` |
| Сетевое обнаружение | `netsh advfirewall firewall show rule name=...` |
| SMB v2/v3 | `Get-SmbServerConfiguration` |

Кнопка **«Обновить диагностику»** перезапускает все проверки в фоновом потоке.

## FAQ

**Q: Программа говорит «нет прав администратора».**  
A: Нажмите «Да» в диалоге UAC или запустите через `Запустить_ShareWizard.bat`.

**Q: Ошибка «New-SmbShare: Access is denied».**  
A: Убедитесь, что служба LanmanServer запущена (индикатор в диагностике).

**Q: Пользователи не видят ресурс в сетевом окружении.**  
A: Включите «Сетевое обнаружение» и убедитесь, что оба компьютера в одной подсети.

**Q: Windows 7-клиент не может подключиться.**  
A: Отключите SMB-шифрование (оно несовместимо с SMB 2.0).

**Q: Как удалить ресурс?**  
A: `Remove-SmbShare -Name "ИМЯ" -Force` в PowerShell от администратора.

**Q: Как проверить, что ресурс создан?**  
A: `Get-SmbShare` в PowerShell покажет все активные ресурсы.
